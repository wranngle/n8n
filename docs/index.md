## Core Concepts

### Hook-Driven Architecture

This repository implements a **deterministic hook-driven architecture** where:

1. **Hooks intercept user prompts** and route to appropriate skills
2. **Skills contain domain expertise** in modular, reusable form
3. **The 21-step protocol** ensures consistent, high-quality workflow output

```
User Request → Hook Detection → Skill Invocation → Protocol Execution → Validated Output
```

### Workflow Governance (DEV-Only Policy)

| Phase | Modifiable | Description |
|-------|------------|-------------|
| **DEV** | ✅ | All active development |
| **ARCHIVED** | ❌ | Deprecated, read-only |

- Deletion is BLOCKED (archive instead)
- New workflows auto-tagged as DEV
- Only DEV workflows are modifiable
